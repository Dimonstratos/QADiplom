### Задача:

## API-тесты
Необходимо проверить, что при обращении к сервису через API, он отдает те же данные. что и при UI-тестах. Такую проверку
можно провести с помощью четырех тестов, которые будут проверять статус карт в БД.

#### Сценарии:
1. Оплата по карте 4444 4444 4444 4441 (http://localhost:8080) ответ сервера - 200 ОК, статус карты - APPROVED
2. Оплата по карте 4444 4444 4444 4442 (http://localhost:8080) ответ сервера - 400 (Bad request)
3. Отправка зявки на кредит по карте 4444 4444 4444 4441 (http://localhost:8080) ответ сервера - 200 ОК, статус карты - APPROVED
4. Отправка зявки на кредит по карте 4444 4444 4444 4442 (http://localhost:8080) ответ сервера - 400 (Bad request)

## 3. Перечень и описание возможных рисков при автоматизации
1. Сложность с настройкой автотестов.
2. Сложности с запуском и настройкой SUT.
3. Правка, изменение дизайна, html, css со временем приведет к неактуальности данных автотестов.
## 4. Интервальная оценка с учетом рисков (в часах)
1. Планирование автоматизации тестирования - 7-8 часов.
2. Написание тестов - 60-70 часов.
3. Подготовка отчетов по итогам автоматизированного тестирования 14-16 часов.
4. Подготовка отчетов по итогам автоматизации - 7-8 часов.
## 5. План сдачи дипломной работы
1. Подготовка авто-тестов - 10 рабочих дней после утверждения плана тестирования.
2. Подготовка отчета по результатам тестирования - 5 рабочих дней после прогона тестов.
3. Подготовка отчета по итогам автоматизации - 5 рабочих дней.
